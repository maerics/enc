# ed25519

Ed25519 has no standard encryption scheme, so unlike `rsa` there is **no**
top-level encrypt/decrypt — only generate/extract/sign/verify. Keys are
PEM-encoded PKCS8 (private) / PKIX (public), not RSA's PKCS1, so they are
much smaller.

## ed25519 generate (alias: gen)

`--private-key`/`--public-key` output filenames, default `-` (stdout).

## ed25519 extract (aliases: extract-public-key, extract-public, epk, ep, e)

`--private-key` default `-` (stdin), `--public-key` default `-` (stdout).

## ed25519 sign / ed25519 verify

- `sign`: reads stdin, writes raw 64-byte signature to stdout.
  `--private-key`/`-k/--key`. **No `--hash` flag** — Ed25519 always hashes
  internally and it isn't user-selectable (unlike `rsa sign`).
- `verify`: reads stdin, checks against `-s/--signature string` (required)
  using `--public-key`/`-k/--key`. Success: writes input back unchanged.
  Failure: exits non-zero, writes nothing.

Known gaps: OpenSSH key format and raw key input are not supported (PEM
PKCS8/PKIX only) — see [../known-gaps.md](../known-gaps.md).
